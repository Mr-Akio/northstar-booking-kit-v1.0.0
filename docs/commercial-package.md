# Commercial Package Notes

This document describes the intended packaging for a sellable `Commercial V1` release of Northstar Booking Kit.

## Positioning

Recommended product title:

`Northstar Booking Kit — Production Booking Starter for Time-Based Businesses`

Recommended subtitle:

`A reusable full-stack booking system starter for appointments, rooms, rentals, and reservable resources.`

## Recommended pricing structure

These are packaging recommendations, not legal or tax advice.

### Starter
- Single project / single team usage
- Core source code
- Documentation
- Demo seed data

### Pro
- Everything in Starter
- Future updates
- Preset demos
- Higher-touch buyer onboarding docs

### Agency / Extended
- Everything in Pro
- Multi-client usage rights subject to your final license terms
- Best suited for studios, agencies, and internal platforms

## What the repo should prove before sale

- Authentication flow works
- Booking creation works
- Conflict prevention works
- Staff and admin roles work
- Docker and local setup are documented
- Tests, lint, typecheck, and build pass
- Buyer can change branding without large code edits

## Sales assets checklist

- Screenshot set:
  - Home
  - Catalog
  - Resource detail
  - Checkout
  - Customer workspace
  - Admin workspace
- Demo accounts list
- Feature matrix
- Architecture overview
- Deployment notes
- Customization guide
- License notes

## Feature matrix ideas

- JWT auth with refresh rotation
- Custom user roles
- Generic resource model
- Availability rules
- Blackout periods
- Booking conflict prevention
- Pending-slot lock configuration
- Customer dashboard
- Staff/admin operations views
- OpenAPI docs
- Docker setup
- CI pipeline
- Demo presets

## Recommended release posture

Commercial V1 should be sold as:
- production-minded
- customizable
- generic across multiple booking businesses

It should not be sold as:
- turnkey multi-tenant SaaS
- payment-complete commerce system
- legally reviewed licensing package without actual legal review
